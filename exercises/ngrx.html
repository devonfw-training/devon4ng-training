ngrx basics

... let's start from scratch

You will discover:

  • Basic setup for @ngrx/store
  • Defining state
  • Dispatching Actions to trigger state transitions
  • Writing a reducer function
  • Testing a reducer function
  • Debugging with the redux devtools

Basic Setup

  • Initialize a new Angular application with Angular CLI
    • ng new ngrx-exercise
  • Add @ngrx/store and @ngrx/store-devtools to the dependencies with npm/yarn
  • Download the Redux Devtools Extension for Chrome, Firefox or Edge

Configure @ngrx I

  • Create root.reducer.ts in src/app
  • Define and export empty interface AppState (ignore warning - we add content later)
  • Define and export an empty object named 'reducers' of type ActionReducerMap<AppState>

Configure @ngrx II

  • Import StoreModule to app.module and configure with root reducer 
    StoreModule.forRoot(reducers)
  • Import StoreDevtoolsModule as well and instrument it 
    StoreDevtoolsModule.instrument()

Check if configured successfully

  • inject Store<AppState> in AppComponent
  • serve application
  • check if redux devtools are initialized

Define number reducer I

  • Create number.reducer.ts in src/app/number
  • Define and export interface NumberState with a single property called value
  • Define and export initialState with value 0

Define number reducer II

  • Define and export 3 Actions without metadata:
    incrementCounter, decrementCounter, resetCounter
  • The actions should increment by 1, decrement by 1 and reset to 0 respectively
  • Define and export the numberReducer 
    function numberReducer(state: NumberState, action: Action): NumberState
  • As return add the createReducer statement to process the action types.

Test number reducer

  • Create number.reducer.spec.ts in src/app/number
  • Create 3 test cases
    1. incrementCounter should return 4 if previous state is 3
    2. decrementCounter should return -2 if previous state is -1
    3. resetCounter should return 0 if previous state is 3

Add number reducer to root reducers

Inside root.reducer.ts

  • Add property number to AppState with value set to NumberState imported from number.reducer
  • Add property number to reducers with value set to reducer imported from number.reducer

Select state from store in AppComponent

Inside app.component.ts

  • Inject the store into your component 
    
                        constructor(private store: Store) {}
  • Create and initialize property numberValue$ with select statement from store 
    
                        numberValue$ = this.store.pipe(select(s => s.number.value));

Display state and dispatch Actions in AppComponent I

Set AppComponent template to 


<button>Increment</button>
<div>Current Number: {{ numberValue$ | async }}</div>
<button>Decrement</button>

<button>Reset Counter</button>

Display state and dispatch Actions in AppComponent II

Inside AppComponent listen for button events and dispatch Actions like for example: 

onResetClick() {
  this.store.dispatch(resetCounter());
}

Check and play

  • Serve application
  • Play with buttons
  • See state transitions in redux devtool

Create login page

  • Create LoginPageComponent in src/app/auth
    • ng g c auth/login-page
  • LoginPageComponent should be a simple form to log in a user

Define typed auth Actions

  • create auth.actions.ts in src/app/auth

  • export two constants

    
export const login = createAction('[authentication] login',
  props<{ username: string, password: string }>());
export const logout = createAction('[authentication] logout');

Create auth reducer I

  • Create auth.reducer.ts in src/app/auth
  • Define and export AuthState with username as a property
  • Create a reducer for the Actions login and logout
  • Login should simply check for the password to be 12345

Create auth reducer II

  • Define and export the initialState and authReducer
  • Register authReducer in root.reducer reducers map

Test auth reducer

  • create auth.reducer.spec.ts in src/app/auth
  • implement tests for both actions

Implement auth logic

  • AppComponent should display LoginPageComponent if no user is logged in
  • AppComponent should display the current user, NumberComponent and a logout button if a user is logged in.
  • Listen for login and logout events and dispatch the proper actions

Check application

  • Serve application and debug using redux-devtools
