  • Current version: 1.5
  • One of the biggest players

AngularJS

Superheroic JavaScript MVW Framework

  • Data binding is fully supported
  • Builtin Routing
  • Uses HTML-Templating language
  • Very good support for tests
  • Clear and nice API for data access
  • Good documentation and very big community
  • Wide tooling support

Angular 2

  • Next Generation Framework for Single Page Applications
  • Modern Javascript (ES6) or Typescript
  • Modern Web Standards (like: Web Components, Fetch Standard, Bluebird Promises, ...)
  • Convention over Configuration
  • ... but easy to change

  • Component based development
  • Separation of Presentation
  • Unobtrusive, small 'footprint' of framework in code
  • Configurable dataflow in databinding (one-way, two-way)

  • Integrates well with .Net (Core)
  • Integrates well with Visual Studio Code
  • Easy to learn for Java / .Net Developers

Which framework to choose?

  • Do you know already what kind of software you want to build?
  • Do you know already which functionalities are the most important for you and your project?
  • Does the team know the framework?
  • Is a framework stable?
  • How difficult is to learn the framework?
  • Is framework good documented?
