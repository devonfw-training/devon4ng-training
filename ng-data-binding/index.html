Data Binding

Data Binding describes how components communicate with each other and how is the data model linked to the html template.

Interpolation


<span>{{ currentBook.author }}</span>
                  

@Component(...)
export class BookEntryComponent {
  currentBook = { author:'John Example', title: 'Angular in a nutshell' };
}
Angular Docs
In the template you have access to class properties inside {{...}}.

Property Binding


<ul>
  <li (click)="selectBook(firstBook)">{{ firstBook.title }}</li>
  <li (click)="selectBook(secondBook)">{{ secondBook.title }}</li>
</ul>
<book-entry [book]="selectedBook"></book-entry >
                  

@Component(...)
export class BookOverviewComponent {
  firstBook = new Book(...);
  secondBook = new Book(...);
  selectedBook: Book|null = null;

  selectBook(book:Book):void {
    this.selectedBook = book;
  }
}
Angular Docs
With property bindings data is passed from parent to child component
Property bindings are always surrounded by [] brackets
The left side of the = is the name of the property as defined by the child.
The right side of the = is the name of the parent component's field to pass to the child.

Property Binding


<ul>
  <li (click)="selectBook(firstBook)">{{ firstBook.title }}</li>
  <li (click)="selectBook(secondBook)">{{ secondBook.title }}</li>
</ul>
<book-entry [book]="selectedBook"></book-entry >
                  

@Component(...)
export class BookEntryComponent {

}
                  

        <div>Title: {{ book?.title }}</div>

Angular Docs
To define a property we add the Input() decorator to the field of a class
The name of the property is defined through the name of the field
The operator in book?.author is for optional property access. So we don't get an error if book is null

Event Binding


<book-entry [currentBook]="books[0]"
           (selectBook)="rentBook($event)">
</book-entry>
<book-entry [currentBook]="books[1]"
           (selectBook)="rentBook($event)">
</book-entry>
                  

@Component(...)
export class BookOverviewComponent {
  books = [new Book(...), new Book(...)];
  rentedBook: Book|null = null;

  rentBook(book: Book) {
    this.rentedBook = book;
  }
}
Angular Docs
With event bindings data is passed from child to parent component
Event bindings are always surrounded by () brackets
The left side of the = is the name of the event as defined by the child.
The right side of the = is the name of the parent component's method to handle the event.
$event is a keyword to pass the events payload as method param

Event Binding


<book-entry [currentBook]="books[0]"
           (selectBook)="rentBook($event)"></book-entry>
            

@Component(...)
export class BookEntryComponent {
  @Input() currentBook: Book;
  @Output() selectBook = new EventEmitter<Book>();

  onSelectBtnClick(): void {
    this.selectBook.emit(this.currentBook);
  }
}
            

    <div>Book title: {{ currentBook.title }}</div>
    <button (click)="onSelectBtnClick()">Rent the book</button>
Angular Docs
To define an event we add the Output() decorator to a field of a class
The name of the event is defined through the name of the field
We assign a EventEmitter to the field. On this EventEmitter we can call emit() to trigger the event. EventEmitter can be parametrized to determine the type of the event's payload.

Input (Data) and Output (Event) Binding

Data flows down
Events flow up

Two way binding


<input type="text" [(ngModel)]="currentBook.author">
                  

@Component(...)
export class BookEntryComponent {
  currentBook = { author:'John Example', title: 'Angular in a nutshell' };
}
Angular Docs
Two way binding is syntactic sure to handle property and event binding simultaniously.
It will immediatly update the view if the value changes in the class and vice versa.
Usually used with <input>
ngModel is a keyword. The right side of the = is the name of the component's field.

Structural Directives

ng-if (Old Syntax)

              
                <div *ngIf="book">
                  <p>{{ book.description }}</p>
                </div>
Angular Docs *ngIf

@if (Angular v17 onwards)

              
                @if(book) 
                {
                  <p>{{ book.description }}</p>
                }

Removes element from the DOM if the book is not present.

Angular Docs @if
Structural directives are marked with a *
Structural directives means they change the structure of the DOM. The other bindings changed values only.
*ngIf takes a condition. If the condition fails, it removes the tag and all of its content

Structural Directives

ng-for (Old Syntax)

            
              <ul>
               <li *ngFor="let myItem of myList">{{ myItem.someProperty }}</li>
              </ul>
Angular Docs *ngIf

@for (Angular v17 onwards)

            
              <ul>
               @for (myItem of myList; track $index) 
               {
                 <li> {{ myItem.someProperty }} </li> 
               }
              </ul>
Angular Docs @if

Creates an element for every item of the collection.

*ngFor creates one node in the DOM for each element in myList
Inside of the tag with ngFor you have access to the loop var as to any other var of the component.

Example - Parent


@Component({
  selector: 'app-book-overview',
  template: `<app-book-entry
                 *ngFor="let book of books"
                 [currentBook]="book"
                 (selectBook)="rentBook($event)">
             </app-book-entry>`
})
export class BookOverviewComponent {
  books = [new Book(...), new Book(...)];
  rentedBook: Book|null = null;

  rentBook(book: Book) {
    this.rentedBook = book;
  }
}

Example - Child

            </div>`

export class BookEntryComponent {

  }
